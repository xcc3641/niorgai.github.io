Android代码混淆

发表于 2014 Dec 26   |   分类于 Android   |  

在这篇文章中说了反编译Android APK的方法，当然作为开发者很多时候不希望某些涉及机密的代码被别人看到，于是需要在打包APK的时候做一些代码混淆。

混淆步骤：

  1. 在项目的project.properties文件中一般已经默认注释了混淆的代码，代码为

    proguard.config=${sdk.dir}/tools/proguard/proguard-android.txt:proguard-project.txt

    这是利用Android自带sdk的规则来混淆代码的，如果需要根据自己的规则来混淆代码，需要这样做：

    1. 在项目中新建一个名为proguard.cfg的文件,并向其中添加自定义的规则。

    2. 在project.properties添加以下代码：

      proguard.config=proguard.cfg

      即将混淆的规则制定为proguard.cfg文件。

    3. 按照正常流程打包APK即可，同时会在项目中生成proguard文件夹。

这是一些混淆的规则(其他的可以到这里查询)：

-keepattributes Signature

//不进行混淆保持原样 
-keep public class * extends android.app.Activity 

//保护指定的类和类的成员的名称，如果所有指定的类成员出席 
-keepclasseswithmembernames class * { 
    native <methods>; 
} 

//保护指定的类和类的成员，但条件是所有指定的类和类成员是要存在 
-keepclasseswithmembers class * { 
    public <init>(android.content.Context, android.util.AttributeSet); 
} 

//保护指定类的成员，如果此类受到保护他们会保护的更好 
-keepclassmembers class * extends android.app.Activity { 
       public void *(android.view.View); 
} 

//保护指定的类文件和类成员 
-keep class * implements android.os.Parcelable { 
     public static final android.os.Parcelable$Creator *; 
}

如果在打包的过程中出现以下错误：

  1. Proguard returned with error code 1. See console

    解决方法为：在Proguard官网下载最新的压缩包，解压后将lib和bin文件夹替换到sdk/tool/proguard文件夹中。
