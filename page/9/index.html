Android相册选取图片

目前从图库选择图片可以有两种方法：

以下结果属于vivo Xplay,Android Version 4.4.4测试结果，如果有其他机型测试我会补充
1.Intent.ACTION_PICK：

Intent intent = new Intent(Intent.ACTION_PICK,
            android.provider.MediaStore.Images.Media.EXTERNAL_CONTENT_URI);
startActivityForResult(intent, PHOTO_PICK);

onActivityResult返回的Intent中：

  • intent.getData()会返回所选图片的Uri，格式为content://media/external/images/media/919
  • intent.getExtras()返回的bundle中包含一个boolean变量，通过data.getExtras().get("isFinishGallery")获取,应该是指是否获取完毕，查询文档没有官方结果。
  • intent.putExtra("return-data",false)对返回值没有影响。

2.Intent.ACTION_GET_CONTENT

Intent intent = new Intent(Intent.ACTION_GET_CONTENT);
intent.setType("image/*");//不加会导致crash

onActivityResult返回的Intent中：

  • intent.getData()返回所选图片的Uri，格式如下：
    • 系统版本大于等于19(KitKat):content://com.android.providers.media.documents/document/image%3A919
    • 系统版本低于19:content://media/external/images/media/919

关于返回Uri格式的说明：

  1. content://media/external/images/media/919格式的Uri，可以通过以下函数获取真实路径：

    private String getRealPathFromURI(Context context,Uri contentUri) {
    String[] proj = {MediaStore.Images.Media.DATA};
    Cursor cursor = context.getContentResolver().query(contentUri, proj, null, null, null);
    if (cursor == null)
        return null;
    cursor.moveToFirst();
    int columnIndex = cursor.getColumnIndex(proj[0]);
    String path = cursor.getString(columnIndex);
    cursor.close();
    return path;
}
  2. Uri获取bitmap的方法可以参考Bitmap相关:读取大图片.

Git小技巧

add：

  1. git add -u 只添加更新了的文件

commit：

  1. commit —amend会修正上次的提交(包括description)
  2. 直接git commit会通过vim打开，可以编辑详细的commit信息。

  3. git rebase 可以理解为重新定位base。

    比如现在从master的A点开了一个分支branch，branch改动了，但是master也改动到了B。那么可以这样做：

    git checkout master; 
git pull origin master; 
git checkout branch; 
git rebase master;

    这样branch也是从master的B点为base了。可能会遇到冲突，打开相应文件更改为正确的内容然后重新add修改后的文件，继续执行git rebase --continue即可。

  4. git rebase -i 从某一个tag开始重新整理commit点。

    eg：如果当前commit是A -> B ->C ->D,想调整为A->C->B->D.

    1. git rebase -i A,即从B点开始调整，这时候会用自带的编辑器打开.git/rebase-merge/git-rebase-todo文件，内容如下：(如果遇到提示:Could not execute editor,表示没有找到文本编辑器，因为我用的是vim，所以直接添加vim路径到git的config中：git config --global core.editor "/usr/bin/vim",然后重新运行rebase命令即可)

    2. 打开后文件前部分内容大致如下：

      pick 2c97b26 A
pick fd19f8e B
pick 02849bf C
pick bd73d4d D

# Rebase 0580eab..74d8a3d onto 0580eab
# ...[chunked]

      文件中第一列表示操作指令：

      pick = 使用这个commit ，不改动
reword = 使用这个commit ，但改动commit message
edit = 使用这个commit，但改动commit 的內容
squash = 使用这个commit，但会与上面的commit合并，commit的message也会合并
fixup = squash + 只使用上面commit的message
exec = 执行一条命令

      上面四条commit信息可以随意移动位置，复制粘贴即可。

      3.:wq退出编辑器,会自动继续rebase操作，最后成功整理commit信息。

push：

  1. push -f强制推送本地代码

merge:

  1. git merge —squash，merge but do not commit

撤销

  1. git rm —cache file会将已经提交的file从git仓库删除，并重置文件为untracked状态
  2. git clean -f 删除 untracked files
  3. git clean -fd 连 untracked 的目录也一起删掉

  4. git clean -xfd 连 gitignore 的untrack 文件/目录也一起删掉 （慎用，一般这个是用来删掉编译出来的 .class之类的文件用的）

    在用上述 git clean 前，加上 -n 参数来先看看会删掉哪些文件，防止重要文件被误删

    1. git clean -nxfd
    2. git clean -nf
    3. git clean -nfd

branch

  1. git push origin :branch-name，推一个空的分支到server上，即删除该分支。
  2. git branch -r -d origin/branch-name删除本地分支

简单使用AsyncTask实现异步执行任务

包android.os.AsyncTask用于执行异步任务，使用AsyncTask必须继承AsyncTask类，并传入三个泛型参数：

Params: 新建任务时的参数。
Progress: 任务在后台执行的进度值类型，一般是Integer。
Result: 任务执行后的返回值类型。

同时必须实现doInBackground方法，顾名思义执行后台任务。

AsyncTask还有一些方法：

//任务开始前调用
protected void onPreExecute()

//后台任务执行,Params泛型数组作为参数传入，Result泛型作为返回值。
protected Result doInBackground(Params... params)

//通过AsyncTask.publishProgress调用，Progress泛型数组会当做参数传入。
protected void onProgressUpdate(Progress... values) 

//任务运行后(doInBackground方法结束后)调用，doInBackground方法的返回值作为参数。
protected void onPostExecute(Result result)

//通过AsyncTask.cancel方法调用，doInBackground方法的返回值作为参数。
//先调用cancel方法1，再调用cancel方法2.
1.protected void onCancelled(Result result)
2.protected void onCancelled()

在使用AsyncTask时，除了doInBackground方法，其他方法都可以访问UI控件。

使用AsyncTask：

//AsyncTask定义
class MyTask extends AsyncTask<String,Integer,Long>
//新建AsyncTask
MyTask myTask = new MyTask();
myTask.execute("task1","task2");

取消AsyncTask：

//通过mayInterruptIfRunning参数决定是否会在任务运行时结束任务。
myTask.cancel(boolean mayInterruptIfRunning);
