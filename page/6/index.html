Git小技巧

发表于 2015 Jan 26   |   分类于 方便开发   |  

add：

  1. git add -u 只添加更新了的文件

commit：

  1. commit —amend会修正上次的提交(包括description)
  2. 直接git commit会通过vim打开，可以编辑详细的commit信息。

  3. git rebase 可以理解为重新定位base。

    比如现在从master的A点开了一个分支branch，branch改动了，但是master也改动到了B。那么可以这样做：

    git checkout master; 
git pull origin master; 
git checkout branch; 
git rebase master;

    这样branch也是从master的B点为base了。可能会遇到冲突，打开相应文件更改为正确的内容然后重新add修改后的文件，继续执行git rebase --continue即可。

  4. git rebase -i 从某一个tag开始重新整理commit点。

    eg：如果当前commit是A -> B ->C ->D,想调整为A->C->B->D.

    1. git rebase -i A,即从B点开始调整，这时候会用自带的编辑器打开.git/rebase-merge/git-rebase-todo文件，内容如下：(如果遇到提示:Could not execute editor,表示没有找到文本编辑器，因为我用的是vim，所以直接添加vim路径到git的config中：git config --global core.editor "/usr/bin/vim",然后重新运行rebase命令即可)

    2. 打开后文件前部分内容大致如下：

      pick 2c97b26 A
pick fd19f8e B
pick 02849bf C
pick bd73d4d D

# Rebase 0580eab..74d8a3d onto 0580eab
# ...[chunked]

      文件中第一列表示操作指令：

      pick = 使用这个commit ，不改动
reword = 使用这个commit ，但改动commit message
edit = 使用这个commit，但改动commit 的內容
squash = 使用这个commit，但会与上面的commit合并，commit的message也会合并
fixup = squash + 只使用上面commit的message
exec = 执行一条命令

      上面四条commit信息可以随意移动位置，复制粘贴即可。

      3.:wq退出编辑器,会自动继续rebase操作，最后成功整理commit信息。

push：

  1. push -f强制推送本地代码

merge:

  1. git merge —squash，merge but do not commit

撤销

  1. git rm —cache file会将已经提交的file从git仓库删除，并重置文件为untracked状态
  2. git clean -f 删除 untracked files
  3. git clean -fd 连 untracked 的目录也一起删掉

  4. git clean -xfd 连 gitignore 的untrack 文件/目录也一起删掉 （慎用，一般这个是用来删掉编译出来的 .class之类的文件用的）

    在用上述 git clean 前，加上 -n 参数来先看看会删掉哪些文件，防止重要文件被误删

    1. git clean -nxfd
    2. git clean -nf
    3. git clean -nfd

branch

  1. git push origin :branch-name，推一个空的分支到server上，即删除该分支。
  2. git branch -r -d origin/branch-name删除本地分支

简单使用AsyncTask实现异步执行任务

发表于 2015 Jan 16   |   分类于 Android   |  

包android.os.AsyncTask用于执行异步任务，使用AsyncTask必须继承AsyncTask类，并传入三个泛型参数：

Params: 新建任务时的参数。
Progress: 任务在后台执行的进度值类型，一般是Integer。
Result: 任务执行后的返回值类型。

同时必须实现doInBackground方法，顾名思义执行后台任务。

AsyncTask还有一些方法：

//任务开始前调用
protected void onPreExecute()

//后台任务执行,Params泛型数组作为参数传入，Result泛型作为返回值。
protected Result doInBackground(Params... params)

//通过AsyncTask.publishProgress调用，Progress泛型数组会当做参数传入。
protected void onProgressUpdate(Progress... values) 

//任务运行后(doInBackground方法结束后)调用，doInBackground方法的返回值作为参数。
protected void onPostExecute(Result result)

//通过AsyncTask.cancel方法调用，doInBackground方法的返回值作为参数。
//先调用cancel方法1，再调用cancel方法2.
1.protected void onCancelled(Result result)
2.protected void onCancelled()

在使用AsyncTask时，除了doInBackground方法，其他方法都可以访问UI控件。

使用AsyncTask：

//AsyncTask定义
class MyTask extends AsyncTask<String,Integer,Long>
//新建AsyncTask
MyTask myTask = new MyTask();
myTask.execute("task1","task2");

取消AsyncTask：

//通过mayInterruptIfRunning参数决定是否会在任务运行时结束任务。
myTask.cancel(boolean mayInterruptIfRunning);

Android线程相关

发表于 2015 Jan 14   |   分类于 Android   |  

Android经常会需要处理一些耗时操作(比如下载，上传等)，执行这些耗时操作的时候，UI是没有反应的，如果时间很久会报ANR，所以需要开启新线程来执行耗时操作。

正确的开启新线程的方法：

new Thread(new Runnable() {
    @Override
    public void run() {
        //此处进行耗时操作
    }
}).start();

Android中Activity所在窗口的线程叫做主线程(Main Thread),因此所有在窗口上显示的控件(UI控件)都是在主线程中创建的，而在主线程创建的新线程中是不允许访问非当前线程(主要是主线程)创建的控件的。所以才会有很多Android坑是不能在非UI线程中访问UI控件。

通常会有以下几种解决方法：

  1. View.post方法

    view.post(new Runnable() {
    @Override
    public void run() {
        //对该view进行操作
    }
});

  2. Activity.runOnUiThread方法

    MainActivity.this.runOnUiThread(new Runnable() {
    @Override
    public void run() {
        //该方法如果在UI线程中执行，Runnale.run方法会立即执行
        //如果在非UI线程中执行，系统会将该方法的引用加入UI线程的任务队列中
    }
});

  3. 使用android.os.handler对象，handler将访问UI线程的代码的引用压入UI线程的处理队列，在其他代码执行完后执行handleMessage方法。

    Message message = new Message();  //Message对象可以放一些参数
handler.sendMessage(message);    //sendMessage方法通知handle

Handler handler = new Handler(){
    @Override
    public void handleMessage(Message msg) {
        //可以在这里进行UI操作
        super.handleMessage(msg);
    }

};

以上总结了开启新线程的方法和进行UI操作的方法，那么综合起来比较好的解决方法是：

  1. 先定义Handler对象

    private Handler handler = null;

  2. 然后创建新线程，并在线程结束时进行调用hanlder.sendMessage方法。

    new Thread(new Runnable() {
    @Override
    public void run() {
        //此处进行耗时操作
        handler.sendMessage(msg);
    }
}).start();

  3. 在handler.handleMessage方法中进行UI操作。

    handler = new Handler(){
    @Override
    public void handleMessage(Message msg) {
        //可以在这里进行UI操作
        super.handleMessage(msg);
    }

};

Android序列化对象

发表于 2015 Jan 12   |   分类于 Android   |  

默认情况下，Activity之间只能传递基本数据类型，如果需要传递某个对象，则需要将其序列化，Android中序列化便是实现Serializable或则Parcelable接口。

序列化:永久性保存对象,即保存对象的字节序列到本地文件中。

Serializable的作用是将对象的属性保存到本地文件、数据库、网络流以方便传输。Parcelable的设计初衷是因为Serializable的效率太慢，为了**在程序内不同组件间及不同Android程序间(AIDL)高效传输而设计。

使用的选择：

  1. 在内存间传递数据时，推荐使用Parcelable，因为其内存开销小，性能高。
  2. 在需要保存时，推荐使用Serializable，因为Parcelable不适合做持久化存储。
  3. Serializable使用IO读写存储在硬盘上，而Parcelable是直接在内存中读写，很明显内存的读写速度通常大于IO读写，所以在Android中通常优先选择Parcelable。

接下来利用一个例子说明如何实现这些接口。首先是初始对象：

public class Test {
    int num;
    String string;
}

如果实现Serializable接口，只需要继承java.io.Serializable，并提供一个serialVersionUID即可。

public class Test implements Serializable {

    final long serialVersionUID = 12345678l;
    int num;
    String string;
}

如果实现Parcelable,需要继承android.os.Parcelable，重写describeContents方法和writeToParcel方法，并定义一个Parcelable.Creator类型的静态变量，在该静态变量中实现readFromParcel方法(自己定义的名字)。

注意：

  1. writeToParcel方法中将变量按顺序写入Parcel

  2. Parcelable.Creator的createFromParcel方法，从Parcel读入变量时必须按照writeToParcel方法中的顺序读入。

    public class Test implements Parcelable{
    int num;
    String string;

    @Override
    public int describeContents() {
            return 0;
        }

        @Override
        public void writeToParcel(Parcel dest, int flags) {
            dest.writeInt(num);
             dest.writeString(string);
        }

    public static Creator<Test> creator = new Creator<Test>() {
            @Override
        public Test createFromParcel(Parcel source) {
            return new Test(source);
        }

        @Override
        public Test[] newArray(int size) {
            return new Test[size];
        }
    };

    Test(){}

    Test(Parcel source){
                readFromParcel(source);
        }

    private void readFromParcel(Parcel source){
        //读取对象数据，必须按照writeToParcel方法中
        //写入数据的顺序依次读取数据
        num = source.readInt();
        string = source.readString();
    }

}

  3. 如果需要序列化自定义的数据，该数据也需要实现Parcelable接口。

    例如希望通过序列化类TestA，类TestA中有一个List存放类TestB。

    class TestA{
    List<TestB> list;
}

    那么TestA与TestB都需要实现Parcelable接口。且类TestA通过parcel.writeTypedList写入list，通过parcel.readTypedList(list,TestB.creator)读取list。

Android Service (三)

发表于 2015 Jan 10   |   分类于 Android   |  

在前面有关服务的第一篇和第二篇中已经说明了怎么开启一个服务，但之前的描述都是指一个应用内的服务，这些服务不能被其他应用程序调用。

Android采用了远程过程调用(Remote Procedure Call,RPC)来使其他应用程序也可以访问本应用程序内的服务。与其他基于RPC的解决方案一样，Android使用接口定义语言(Interface Definition Language)来公开服务的接口，所以跨进程访问的服务便称为AIDL(Android Interface Definition Language)服务。

一般创建AIDL服务的步骤如下(以Ant构建的项目为例)：

  1. 在项目的src文件夹下任意包中新建一个XX.aidl文件，文件内容为相应包名及接口定义。如果aidl文件内容正确，gen目录下回自动生成一个同名的Java接口文件(gradle构建的项目没有找到。。)。
  2. 建立一个Service类并实现aidl文件定义的Java接口。
  3. 在AndroidManiFest.xml文件中注册AIDL服务。

下面用一个例子来介绍：

  1. 新建一个ITest.aidl文件,接口只有一个getData方法。

    package qiu.test

interface ITest{
    String getData();
}

  2. 编写一个TestService类。

    public class TestService extends Service{

    public class TestServiceeImp extends ITest.Stub{

        @Override
         public String getData() throws RemoteException{
            return "Here is data";
        }
    }

    @Override
    public IBinder onBind(Intent intent) {
        return new TestServiceImp();
    }
}

    Stub为IBinder的子类，通过onBind方法返回TestServiceImp对象，客户端便可以获取Service对象。

  3. 在AndroidManiFest.xml文件中配置Service。

    <service android:name=".TestService">
    <intent-filter>
        <action android:name="qiu.test.ITest"/>
    </intent-filter>
</service>

    其实qiu.test.ITest便是客户端用于访问AIDL服务的ID。

以上就是服务端的代码，也可以理解为另一程序的Service代码。下面来编写客户端，或者说调用端的代码。

首先需要将服务端自动生成的ITest.java文件连同包目录一起复制到src文件中。

接下来实现MainActivity类。

public class MainActivity extends Activity {

    @Override
    protected void onCreate(Bundle savedInstanceState) {
        super.onCreate(savedInstanceState);
        setContentView(R.layout.activity_main);

        //获取的服务端Service对象，通过该对象可以调用任意共有方法
        ITest service = null;

        ServiceConnection connection = new ServiceConnection() {
            @Override
            public void onServiceConnected(ComponentName name, IBinder service) {
                //初始化服务端Service对象
                service = ITest.Stub.asInterface(service);
            }

            @Override
            public void onServiceDisconnected(ComponentName name) {

            }
        };

        //绑定服务,使用隐式Intent
        bindService(new Intent("qiu.test.ITest"), connection, BIND_AUTO_CREATE);

        //绑定后就可以调用ITest的任意方法，比如getData。
        System.out.println("Data is : " + service.getData());

    }
}
