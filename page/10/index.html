sharedPreference使用心得

发表于 2014 Dec 6   |   分类于 Android   |  

sharedPreference非常适合key-value形式的数据存储，它会将这些字段一起以文件的形式保存在本地，是一种比较可靠的数据保存方式。

存储位置为：

/data/data/<package name>/shared_prefs

使用方法如下：

name字段为自己定义的String，实际上便是本地保存所保存数据的文件名(不包括拓展名)。
如这里便是

/data/data/<package name>/shared_prefs/name.xml

读：

public static boolean getLogin(Context context){  
    SharedPreferences preferences = context.getSharedPreferences(name,Context.MODE_PRIVATE);  
    return preferences.getBoolean("login", false);  
}

写：

public static void setLogin(Context context, boolean login){  
    SharedPreferences preferences = context.getSharedPreferences(name,Context.MODE_PRIVATE);  
    SharedPreferences.Editor editor = preferences.edit();  
    editor.putBoolean("login",login);  
    editor.commit();  //同步
    editor.apply();     //异步
}

SharedPreferences.Editor提供了两种提交的方法，分别是commit和apply，两种方法都是原子过程(即改过程不可拆分，如同a＝0)，但两种方法有区别：

  1. commit会同步提交数据到磁盘，同时给出boolean返回值代表是否成功。多个并发的commit同时执行时，commit会等待正在运行的commit运行完毕再运行，从而较低效率。
  2. apply先将数据提交到内存，再异步提交到磁盘，apply并发操作时后面的apply会覆盖前面的apply，提高效率，但apply方法没有返回值，因此不会有错误的提示。

注意写的方式不能这么写：

public static void setLogin(Context context, boolean login){  
    SharedPreferences preferences = context.getSharedPreferences(name,Context.MODE_PRIVATE);  
    preferences.edit().putBoolean("login",login);  
    preferences.edit().commit();  
}

Android布局心得

发表于 2014 Dec 5   |   分类于 Android开发tips   |  

布局最最常用的无非就是linearLayout和RelativeLayout。

linearLayout好处是渲染速度快，性能最好。

RelativeLayout好处是实现布局的嵌套比较少，实现某些特定布局比较方便。

之前写界面的时候，往往给的UI图都是充满整个界面的，所以习惯性的用RelativeLayout或者LinearLayout去布局，利用align或者weight来设置高度。

如：

<RelativeLayout xmls:android="XXXXXXX"
    ....>
    <your layout>
</RelativeLayout>

但是往往界面做完后，对一些手机的适配很差（特别是小屏幕的手机），所以觉得这样布局很有问题。

于是就要考虑改进了。。

其实可以利用ScrollView，然后内部布局使用固定高度。因为ScrollView只能有一个直接子View，所以布局变成这样。

<RelativeLayout xmls:android="XXXXX"
    ....>
    <ScrollView
        ....>
        <RelativeLayout                     !-- 这是之前的根layout --!
            ...>
            <your layout>
        </RelativeLayout>
    </ScrollView>
</RelativeLayout>

注意不要把Title加到ScrollView里面了~~

style里面android:windowBackground可以设置全局的背景颜色, 也不用担心过度渲染.

  1. ImageView的android:adjustViewBounds属性可以使图片正好贴合边框.
  2. 如果希望在RelativeLayout中放置一个背景图, 同时RelativeLayout的高度已背景图为准.如果简单地将图片设置RelativeLayout的背景,会导致RelativeLayout上下都有padding(暂时不知道为什么…).
    同时如果设置了RelativeLayout的高度为wrap_content后会导致below和above属性失效…..
    目前想到的解决方法是外层包一个wrap_content的FrameLayout, 里面同时放置一个ImageView和fill_parent的RelativeLayout.
