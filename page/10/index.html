Android序列化对象

默认情况下，Activity之间只能传递基本数据类型，如果需要传递某个对象，则需要将其序列化，Android中序列化便是实现Serializable或则Parcelable接口。

序列化:永久性保存对象,即保存对象的字节序列到本地文件中。

Serializable的作用是将对象的属性保存到本地文件、数据库、网络流以方便传输。Parcelable的设计初衷是因为Serializable的效率太慢，为了**在程序内不同组件间及不同Android程序间(AIDL)高效传输而设计。

使用的选择：

  1. 在内存间传递数据时，推荐使用Parcelable，因为其内存开销小，性能高。
  2. 在需要保存时，推荐使用Serializable，因为Parcelable不适合做持久化存储。
  3. Serializable使用IO读写存储在硬盘上，而Parcelable是直接在内存中读写，很明显内存的读写速度通常大于IO读写，所以在Android中通常优先选择Parcelable。

接下来利用一个例子说明如何实现这些接口。首先是初始对象：

public class Test {
    int num;
    String string;
}

如果实现Serializable接口，只需要继承java.io.Serializable，并提供一个serialVersionUID即可。

public class Test implements Serializable {

    final long serialVersionUID = 12345678l;
    int num;
    String string;
}

如果实现Parcelable,需要继承android.os.Parcelable，重写describeContents方法和writeToParcel方法，并定义一个Parcelable.Creator类型的静态变量，在该静态变量中实现readFromParcel方法(自己定义的名字)。

注意：

  1. writeToParcel方法中将变量按顺序写入Parcel

  2. Parcelable.Creator的createFromParcel方法，从Parcel读入变量时必须按照writeToParcel方法中的顺序读入。

    public class Test implements Parcelable{
    int num;
    String string;

    @Override
    public int describeContents() {
            return 0;
        }

        @Override
        public void writeToParcel(Parcel dest, int flags) {
            dest.writeInt(num);
             dest.writeString(string);
        }

    public static Creator<Test> creator = new Creator<Test>() {
            @Override
        public Test createFromParcel(Parcel source) {
            return new Test(source);
        }

        @Override
        public Test[] newArray(int size) {
            return new Test[size];
        }
    };

    Test(){}

    Test(Parcel source){
                readFromParcel(source);
        }

    private void readFromParcel(Parcel source){
        //读取对象数据，必须按照writeToParcel方法中
        //写入数据的顺序依次读取数据
        num = source.readInt();
        string = source.readString();
    }

}

  3. 如果需要序列化自定义的数据，该数据也需要实现Parcelable接口。

    例如希望通过序列化类TestA，类TestA中有一个List存放类TestB。

    class TestA{
    List<TestB> list;
}

    那么TestA与TestB都需要实现Parcelable接口。且类TestA通过parcel.writeTypedList写入list，通过parcel.readTypedList(list,TestB.creator)读取list。

Android DebugUtils工具类

开发过程中经常需要一些调试信息，与其每次都自己打Log或者System.out.println，不如实现写好一个DebugUtils类，即方便调试又可以在正式发布的时候直接设置DebugFlag为false，而不用删除调试信息。

首先介绍一些Log类，android.util.Log常用的方法有以下：

Log.v,Log.d,Log.i,Log.w,Log.e以及Log.wtf。

根据首字母对应Verbose,Debug,Info, Warn，Error和What the f**k….

  • Log.v : Verbose为啰嗦的意思，如果希望查看每一个输出，使用Log.v
  • Log.d : 主要用于Debug信息。
  • Log.i : 主要用于输出拥有信息，比如成功连接到服务器~
  • Log.w : 当有些问题不确定，但又不算是bug时，使用Log.w
  • Log.e : 用于捕捉已知Error.
  • Log.wtf : 这个算是题外话了。用于捕捉你特别不想它出现的重大bug.what the f**k…

以下就是LogUtils类,出处为这篇博客：

public class LogUtils{
    public static final String TAG = "Debug";
    private static final boolean DEBUG_FLAG = true;

    public static void println( String printInfo ){
        if( DEBUG_FLAG && null != printInfo ){
            System.out.println( printInfo );
        }
    }

    public static void print( String printInfo ){
        if( DEBUG_FLAG && null != printInfo ){
            System.out.print( printInfo );
        }
    }

    public static void printLogI( String logInfo ){
        printLogI( TAG, logInfo );
    }

    public static void printLogI( String tag, String logInfo ){
        if( DEBUG_FLAG && null != tag && null != logInfo ){
            Log.i(tag, logInfo);
        }
    }

    public static void printLogE( String logInfo ){
        printLogE( TAG, logInfo );
    }

    public static void printLogE( String tag, String logInfo ){
        if( DEBUG_FLAG && null != tag && null != logInfo ){
            Log.e( tag, logInfo );
        }
    }

    public static void printLogW( String logInfo ){
        printLogW( TAG, logInfo );
    }

    public static void printLogW( String tag, String logInfo ){
        if( DEBUG_FLAG && null != tag && null != logInfo ){
            Log.w( tag, logInfo );
        }
    }

    public static void printLogD( String logInfo ){
        printLogD( TAG, logInfo );
    }

    public static void printLogD( String tag, String logInfo ){
        if( DEBUG_FLAG && null != tag && null != logInfo ){
            Log.d( tag, logInfo );
        }
    }

    public static void printLogV( String logInfo ){
        printLogV( TAG, logInfo );
    }

    public static void printLogV( String tag, String logInfo ){
        if( DEBUG_FLAG && null != tag || null != logInfo ){
            Log.v( tag, logInfo );
        }
    }

    public static void printLogWtf( String logInfo ){
        printLogWtf( TAG, logInfo );
    }

    public static void printLogWtf( String tag, String logInfo ){
        if( DEBUG_FLAG && null != tag && null != logInfo ){
            Log.wtf( tag, logInfo );
        }
    }

    public static void showToast( Context context, String toastInfo ){
        if( null != context && null != toastInfo ){
            Toast.makeText(context, toastInfo, Toast.LENGTH_LONG).show( );
        }
    }

    public static void showToast( Context context, String toastInfo, int timeLen ){
        if( null != context && null != toastInfo && ( timeLen > 0 ) ){
            Toast.makeText( context, toastInfo, timeLen ).show( );
        }
    }

}

Android Service (三)

在前面有关服务的第一篇和第二篇中已经说明了怎么开启一个服务，但之前的描述都是指一个应用内的服务，这些服务不能被其他应用程序调用。

Android采用了远程过程调用(Remote Procedure Call,RPC)来使其他应用程序也可以访问本应用程序内的服务。与其他基于RPC的解决方案一样，Android使用接口定义语言(Interface Definition Language)来公开服务的接口，所以跨进程访问的服务便称为AIDL(Android Interface Definition Language)服务。

一般创建AIDL服务的步骤如下(以Ant构建的项目为例)：

  1. 在项目的src文件夹下任意包中新建一个XX.aidl文件，文件内容为相应包名及接口定义。如果aidl文件内容正确，gen目录下回自动生成一个同名的Java接口文件(gradle构建的项目没有找到。。)。
  2. 建立一个Service类并实现aidl文件定义的Java接口。
  3. 在AndroidManiFest.xml文件中注册AIDL服务。

下面用一个例子来介绍：

  1. 新建一个ITest.aidl文件,接口只有一个getData方法。

    package qiu.test

interface ITest{
    String getData();
}

  2. 编写一个TestService类。

    public class TestService extends Service{

    public class TestServiceeImp extends ITest.Stub{

        @Override
         public String getData() throws RemoteException{
            return "Here is data";
        }
    }

    @Override
    public IBinder onBind(Intent intent) {
        return new TestServiceImp();
    }
}

    Stub为IBinder的子类，通过onBind方法返回TestServiceImp对象，客户端便可以获取Service对象。

  3. 在AndroidManiFest.xml文件中配置Service。

    <service android:name=".TestService">
    <intent-filter>
        <action android:name="qiu.test.ITest"/>
    </intent-filter>
</service>

    其实qiu.test.ITest便是客户端用于访问AIDL服务的ID。

以上就是服务端的代码，也可以理解为另一程序的Service代码。下面来编写客户端，或者说调用端的代码。

首先需要将服务端自动生成的ITest.java文件连同包目录一起复制到src文件中。

接下来实现MainActivity类。

public class MainActivity extends Activity {

    @Override
    protected void onCreate(Bundle savedInstanceState) {
        super.onCreate(savedInstanceState);
        setContentView(R.layout.activity_main);

        //获取的服务端Service对象，通过该对象可以调用任意共有方法
        ITest service = null;

        ServiceConnection connection = new ServiceConnection() {
            @Override
            public void onServiceConnected(ComponentName name, IBinder service) {
                //初始化服务端Service对象
                service = ITest.Stub.asInterface(service);
            }

            @Override
            public void onServiceDisconnected(ComponentName name) {

            }
        };

        //绑定服务,使用隐式Intent
        bindService(new Intent("qiu.test.ITest"), connection, BIND_AUTO_CREATE);

        //绑定后就可以调用ITest的任意方法，比如getData。
        System.out.println("Data is : " + service.getData());

    }
}
