Android SQLite数据库

发表于 2015 Jan 3   |   分类于 Android   |  

操作数据库主要利用android.database.sqlite.SQLiteDatabase这个包，使用方法一般有3种：

  1. 使用SQLiteOpenHelper的子类，SQLiteOpenHelper是一个抽象类，其子类必须实现这两个方法：

    方法1：

    @Override
public void onCreate(SQLiteDatabase db) {  

}

    该方法将先在 /data/data//databases 目录下寻找数据库，如果数据库不存在则调用该方法。可以在该方法中调用

        db.execSQL(sql);

方法，其中sql为自定义的sql语句，可以建表及插入数据。

    方法2：

    @Override
public void onUpgrade(SQLiteDatabase db, int oldVersion, int newVersion) {

}

    该方法在数据库存在且当前版本号高于上次版本号时会调用，子类如果不需要可以不进行任何操作。

  2. 如果直接使用SQLiteDatabase可以创建任意路径的数据库，方法原型为：

    public static SQLiteDatabase openOrCreateDatabase(String path, CursorFactory factory)

    然后利用contentValues或者直接调用execSQL等语句直接实现。

  3. 如果只想创建存在于RAM中的数据库，并且在程序退出时自动销毁数据库，可以直接调用

    SQLiteDatabase database = SQLiteDatabase.create(null);

OpenFileOutput & OpenFileIntput 使用

发表于 2014 Dec 30   |   分类于 Android   |  

与sharedPreference相似，OpenFileOutput和OpenFileIntput也是在Context中调用，同时将文件保存在以下路径：

data/data/<package name>/files

name为String类型，表示文件名称

text为String类型，表示所需写入的内容

使用方法(需要注意catch FileNotFoundException)：

写:

OutputStream os = context.openFileOutput(name,Context.MODE_PRIVATE);
os.write(text.getBytes("UTF-8"));
os.close();

读:

InputStream is = context.openFileInput(name);
byte[] buffer = new byte[100];
int byteCount = is.read(buffer);
String str = new String(buffer,0,byteCount,"UTF-8");
is.close();

Android代码混淆

发表于 2014 Dec 26   |   分类于 Android   |  

在这篇文章中说了反编译Android APK的方法，当然作为开发者很多时候不希望某些涉及机密的代码被别人看到，于是需要在打包APK的时候做一些代码混淆。

混淆步骤：

  1. 在项目的project.properties文件中一般已经默认注释了混淆的代码，代码为

    proguard.config=${sdk.dir}/tools/proguard/proguard-android.txt:proguard-project.txt

    这是利用Android自带sdk的规则来混淆代码的，如果需要根据自己的规则来混淆代码，需要这样做：

    1. 在项目中新建一个名为proguard.cfg的文件,并向其中添加自定义的规则。

    2. 在project.properties添加以下代码：

      proguard.config=proguard.cfg

      即将混淆的规则制定为proguard.cfg文件。

    3. 按照正常流程打包APK即可，同时会在项目中生成proguard文件夹。

这是一些混淆的规则(其他的可以到这里查询)：

-keepattributes Signature

//不进行混淆保持原样 
-keep public class * extends android.app.Activity 

//保护指定的类和类的成员的名称，如果所有指定的类成员出席 
-keepclasseswithmembernames class * { 
    native <methods>; 
} 

//保护指定的类和类的成员，但条件是所有指定的类和类成员是要存在 
-keepclasseswithmembers class * { 
    public <init>(android.content.Context, android.util.AttributeSet); 
} 

//保护指定类的成员，如果此类受到保护他们会保护的更好 
-keepclassmembers class * extends android.app.Activity { 
       public void *(android.view.View); 
} 

//保护指定的类文件和类成员 
-keep class * implements android.os.Parcelable { 
     public static final android.os.Parcelable$Creator *; 
}

如果在打包的过程中出现以下错误：

  1. Proguard returned with error code 1. See console

    解决方法为：在Proguard官网下载最新的压缩包，解压后将lib和bin文件夹替换到sdk/tool/proguard文件夹中。

Mac下反编译Android APK

发表于 2014 Dec 23   |   分类于 Android   |  

  1. 所需工具：

    1. JD-GUI,主要反编译Java源代码。

    2. apktool,主要反编译xml等资源文件。需要下载apktool及apktool-install-macosx.

    3. dex2jar,主要将apk中的dex文件转为jar。

  2. 部署：

    1. JD-GUI直接解压就是app了。

    2. apktool需要解压两个压缩包并将”apktool.jar”,”apktool”,”aapt”三个文件放在同一目录下，最好配置一下环境变量。

    3. 解压即可。使用时需要将”d2j-dex2jar.sh”文件及”lib”文件夹一起复制。

  3. 获取xml(使用apktool)：

    1. 使用apktool，命令如下： apktool d “apk的路径” “反编译后的文件路径”。

      eg(我已经cd到apk路径):

      apktool d test.apk temp

    2. 解压后的res路径就是你想要的了。

  4. 获取java(使用dex2jar和JD-GUI)：

    1. 将apk文件的后缀名改为.zip,然后直接解压。会得到一个名为”classes.dex”的文件。

    2. 将”d2j-dex2jar.sh”文件及”lib”文件夹一起复制到”classes.dex”的同一目录下，同时cd到该目录，命令为： d2j-dex2jar.sh “apk的路径” “反编译后的文件路径”。

      eg(我已经cd到该目录下):

      ./d2j-dex2jar.sh classes.dex ./temp

      然后在该目录下会产生一个”classes-dex2jar.jar”的jar文件。

    3. 利用JD-GUI打开该jar文件，所有的代码变一目了然了。可以使用File-Save All Source保存所有的源文件。

Android裁剪图片

发表于 2014 Dec 18   |   分类于 Android   |  

使用Intent：

注意点：

  1. 非官方提供的代码，该方法并不通用，不是所有机器都支持这个intent，不推荐
  2. 裁剪时putExtra设置crop为true即可。
  3. Uri必须以file://开头，不然会导致“无法保存剪裁的图片”的错误。

可以封装这样一段代码：

private void cropImageUri(Uri uri, int outputX, int outputY, int requestCode,Uri outputUri){
    Intent intent = new Intent("com.android.camera.action.CROP");
    intent.setDataAndType(uri, "image/*");
    intent.putExtra("crop", "true");
    intent.putExtra("aspectX", 2);
    intent.putExtra("aspectY", 1);
    intent.putExtra("outputX", outputX);
    intent.putExtra("outputY", outputY);
    intent.putExtra("scale", true);
    intent.putExtra(MediaStore.EXTRA_OUTPUT, outputUri);
    intent.putExtra("return-data", false);
    intent.putExtra("outputFormat", Bitmap.CompressFormat.JPEG.toString());
    intent.putExtra("noFaceDetection", true); // no face detection
    startActivityForResult(intent, requestCode);
}

另外关于putExtra的信号：

附加选项 数据类型 描述
crop String 发送裁剪信号
aspectX int X方向上的比例
aspectY int Y方向上的比例
outputX int 裁剪区的宽
outputY int 裁剪区的高
scale boolean 是否保留比例
return-data boolean 是否将数据保留在Bitmap中返回
data Parcelable 相应的Bitmap数据
circleCrop String 圆形裁剪区域？
MediaStore.EXTRA_OUTPUT (“output”) URI 将URI指向相应的file:///…

    这是基于Gradle的，可以收到加上com.android.camera这个包里面的代码，加上resource即可使用，example里面有使用示例。

  • https://github.com/biokys/cropimage
