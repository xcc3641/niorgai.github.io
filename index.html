一些自适应高度的控件

发表于 2015 Jun 26   |  

很多时候因为业务需求需要一些自适应高度的控件,贴一下自己整理的常见控件~

  1. 自适应高度的ViewPager

    public class WrapContentViewPager extends ViewPager{

    public WrapContentViewPager(Context context) {
        super(context);
    }

    public WrapContentViewPager(Context context, AttributeSet attrs) {
        super(context, attrs);
    }

    @Override
    protected void onMeasure(int widthMeasureSpec, int heightMeasureSpec) {
        int height = 0;
        for(int i = 0; i < getChildCount(); i++) {
            View child = getChildAt(i);
            child.measure(widthMeasureSpec, MeasureSpec.makeMeasureSpec(0, MeasureSpec.UNSPECIFIED));
            int h = child.getMeasuredHeight();
            if(h > height) height = h;
        }

        heightMeasureSpec = MeasureSpec.makeMeasureSpec(height, MeasureSpec.EXACTLY);

        super.onMeasure(widthMeasureSpec, heightMeasureSpec);
    }
}

  2. 自适应高度的ListView(改一下也支持GridView)

    public class WrapContentListView extends ListView{
    public WrapContentListView(Context context) {
        super(context);
    }

    public WrapContentListView(Context context, AttributeSet attrs) {
        super(context, attrs);
    }

    public WrapContentListView(Context context, AttributeSet attrs, int defStyleAttr) {
        super(context, attrs, defStyleAttr);
    }

    @Override
    protected void onMeasure(int widthMeasureSpec, int heightMeasureSpec) {
        int expandSpec = MeasureSpec.makeMeasureSpec(Integer.MAX_VALUE >> 2, MeasureSpec.AT_MOST);
        super.onMeasure(widthMeasureSpec, expandSpec);
    }
}

ProgressBar相关

发表于 2015 May 13   |   分类于 Android   |  

ProgressBar作为一个进度条控件,经常会被用到,但是使用的时候还是有一些注意点的.

  1. 默认的横向Progressbar使用的style是"@android:style/Widget.ProgressBar.Horizontal",但这个style带有圆角,为了去掉这个圆角,我们只需要一直沿着style查看到源码,然后将progressDrawable文件progress_horizontal复制到自己工程的drawable文件下,那换背景色,去掉圆角的都可以随意修改了~.

  2. 注意ProgressBar设置progress的时候要在新线程设置,这个在开发者文档也有提到.

    private Handler mHandler = new Handler();

new Thread(new Runnable() {
            @Override
            public void run() {
                mHandler.post(new Runnable() {
                    @Override
                    public void run() {
                        progressBar.setProgress(newProgress);
                    }
                });
            }
        }).start();

ListView的一些坑

发表于 2015 Apr 24   |   分类于 Android   |  
  1. setEmptyView需要listView与emptyView在一个Layout下，所以需要这样加入（获取ViewParent加入emptyView）：
View view = (LayoutInflater.from(context).inflate(R.layout.empty_goods,null,false));
((ViewGroup)gridView.getParent()).addView(view, new ViewGroup.LayoutParams(ViewGroup.LayoutParams.MATCH_PARENT, ViewGroup.LayoutParams.MATCH_PARENT));
gridView.setEmptyView(view);

  1. ListView setSelection方法有时候可能会失效,原因一个是没有焦点,一个是没有在主线程中执行.解决方法是:

    listView.(new Runnable() {
    @Override
    public void run() {
        listView.requestFocusFromTouch();
        listView.setItemChecked(i, true);
        listView.setSelection(i);
    }
});
