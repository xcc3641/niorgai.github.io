Java接口与抽象类的区别

发表于 2015 Mar 10   |   分类于 Java   |  

  • 抽象类：故名思义，抽象出来的，一般是用于反应子类的通用属性，它不能被实例化，只能被当作子类的父类，继承该抽象类的子类必须实现其抽象方法。

  • 接口：更加抽象化，如果一个类声明了该接口，那么它必须实现这个接口的抽象方法。

对比：

属性 抽象类 接口
构造函数 作为一个类，可以有构造函数 没有
修饰符 作为一个类，正常的修饰都可以 只能是public
多重继承 可以继承一个类和实现多个接口 只能继承一个类或实现多个接口
实现 extends关键字，必须实现所有抽象函数 implements关键字，必须实现全部函数
偶合度 改动抽象类如果给了实现方法，则不用改动继承了它的类 改动接口必须改动实现了该接口的代码

使用场景：

  1. 如果希望方法中有默认实现，使用抽象类
  2. 如果希望实现多重继承，因为Java不支持多继承，所以使用接口
  3. 如果现有功能经常改动，使用抽象类
