Android线程相关

Android经常会需要处理一些耗时操作(比如下载，上传等)，执行这些耗时操作的时候，UI是没有反应的，如果时间很久会报ANR，所以需要开启新线程来执行耗时操作。

正确的开启新线程的方法：

new Thread(new Runnable() {
    @Override
    public void run() {
        //此处进行耗时操作
    }
}).start();

Android中Activity所在窗口的线程叫做主线程(Main Thread),因此所有在窗口上显示的控件(UI控件)都是在主线程中创建的，而在主线程创建的新线程中是不允许访问非当前线程(主要是主线程)创建的控件的。所以才会有很多Android坑是不能在非UI线程中访问UI控件。

通常会有以下几种解决方法：

  1. View.post方法

    view.post(new Runnable() {
    @Override
    public void run() {
        //对该view进行操作
    }
});

  2. Activity.runOnUiThread方法

    MainActivity.this.runOnUiThread(new Runnable() {
    @Override
    public void run() {
        //该方法如果在UI线程中执行，Runnale.run方法会立即执行
        //如果在非UI线程中执行，系统会将该方法的引用加入UI线程的任务队列中
    }
});

  3. 使用android.os.handler对象，handler将访问UI线程的代码的引用压入UI线程的处理队列，在其他代码执行完后执行handleMessage方法。

    Message message = new Message();  //Message对象可以放一些参数
handler.sendMessage(message);    //sendMessage方法通知handle

Handler handler = new Handler(){
    @Override
    public void handleMessage(Message msg) {
        //可以在这里进行UI操作
        super.handleMessage(msg);
    }

};

以上总结了开启新线程的方法和进行UI操作的方法，那么综合起来比较好的解决方法是：

  1. 先定义Handler对象

    private Handler handler = null;

  2. 然后创建新线程，并在线程结束时进行调用hanlder.sendMessage方法。

    new Thread(new Runnable() {
    @Override
    public void run() {
        //此处进行耗时操作
        handler.sendMessage(msg);
    }
}).start();

  3. 在handler.handleMessage方法中进行UI操作。

    handler = new Handler(){
    @Override
    public void handleMessage(Message msg) {
        //可以在这里进行UI操作
        super.handleMessage(msg);
    }

};
